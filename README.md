# 🔬 功效搜尋系統

一個基於 Google Gemini AI 的智能功效分析工具，可分析食物、藥物、保健食品、中藥的功效與交互作用，並提供中醫與西醫的雙重視角分析。

## ✨ 功能特色

### 📝 多項目搜尋
- 支援同時輸入多種食物、藥物、保健食品或中藥
- 用逗號、頓號、換行分隔即可
- 例如：`香蕉、牛奶、維他命C、人參、當歸`

### 📷 圖片辨識
- 上傳食物、藥物、保健食品、中藥的圖片
- AI 自動辨識並分析功效
- 支援 JPG、PNG、WEBP 格式
- 可同時上傳多張圖片

### 🏥 中醫與西醫雙重視角
每個項目都會提供：
- **中醫觀點**：
  - 性味歸經（如：甘、平，歸脾、胃經）
  - 中醫理論中的功效（如：補氣、養血、清熱等）
  - 主治與適應症
  - 中醫配伍禁忌（十八反、十九畏等）
- **西醫觀點**：
  - 現代藥理作用機制
  - 臨床應用與適應症
  - 已知副作用
  - 藥物交互作用
- **綜合建議**：整合中醫與西醫觀點的實用建議

### 🔗 交互作用分析
當輸入多個項目時，系統會分析：
- ✅ **正面效果**：協同作用、增強吸收等
- ⚠️ **負面效果與風險**：
  - 是「絕對禁忌」還是「潛在風險」
  - 作用機理說明（如：單寧酸與蛋白質結合形成胃柿石）
  - 可能產生的症狀
- 🏥 **中醫觀點**：
  - 配伍關係（相須、相使、相畏、相殺、相惡、相反）
  - 中醫禁忌（十八反、十九畏）
  - 中醫角度的使用建議
- 🔬 **西醫觀點**：
  - 現代藥理交互作用機制
  - 臨床研究證據
  - 現代醫學角度的使用建議
- ⏰ **間隔時間建議**：
  - 建議間隔多久食用
  - 為什麼需要這個間隔時間
  - 哪個應該先吃
- 💡 **食用建議**：
  - 搭配其他食物降低風險
  - 特殊處理方式
  - 需要注意的人群
  - 中醫與西醫觀點的整合建議

### 💾 資料儲存
- **暫存功能**：重新整理頁面後結果不會消失
- **禁忌提醒**：可儲存有風險的組合到 IndexedDB，未來搜尋相同組合時會提醒

## 🚀 使用方式

### 1. 設定 API Key
1. 點擊右上角「⚙️ API 設定」
2. 輸入您的 [Google Gemini API Key](https://aistudio.google.com/app/apikey)
3. 點擊「儲存」

### 2. 搜尋功效
1. 在輸入框輸入食物、藥物、保健食品或中藥名稱
2. 或上傳圖片讓 AI 辨識
3. 點擊「🔍 搜尋功效」

### 3. 查看結果
- **單一項目**：直接顯示該項目的詳細分析
- **多個項目**：
  - 上方按鈕可切換查看各項目的詳細分析
  - 下方黃色區塊顯示交互作用分析

## 📋 分析內容

每個項目會顯示：

| 區塊 | 內容 |
|------|------|
| 基本資訊 | 類型、主要成分 |
| 中醫觀點 | 性味歸經、功效、主治、配伍禁忌、使用注意 |
| 西醫觀點 | 藥理作用、臨床應用、副作用、藥物交互作用、使用注意 |
| 綜合建議 | 適用人群、禁忌人群、建議用量、中醫與西醫觀點異同 |

多項目交互作用分析：

| 區塊 | 內容 |
|------|------|
| 正面效果 | 協同作用、增強效果 |
| 負面效果與風險 | 禁忌說明、機理、症狀 |
| 中醫觀點 | 配伍關係、中醫禁忌、中醫建議 |
| 西醫觀點 | 藥理交互作用、臨床研究證據、西醫建議 |
| 間隔時間建議 | 建議間隔時間、原因 |
| 食用建議 | 搭配建議、處理方式、中醫與西醫觀點整合 |

## 🛠️ 技術架構

- **前端**：純 HTML + CSS + JavaScript（單一檔案）
- **AI 模型**：Google Gemini 2.0 Flash
- **資料儲存**：
  - LocalStorage：API Key、暫存結果
  - IndexedDB：禁忌提醒記錄
